# 2026-02-08 — Voice Messages Complete

## What was done

Implemented end-to-end voice message support for Merlin:

1. **transcribe.py** — faster-whisper transcription script, medium model, language=fr, int8 CPU
2. **merlin.py** — Voice message detection, transcription, prompt formatting, emoji flow (🎤→🤔→✅)
3. **Dashboard** — Transcription events logged to structured.jsonl, visible in logs page with duration
4. **Discord UX** — Transcription posted as `> 🎤 *text* (Xs)` in thread before Claude responds

## Iterations during live testing

- Started with `language="en"` — French was phonetically transliterated, not translated. Switched to `language="fr"` for accurate French transcription; Claude handles understanding.
- Started with `base` model — accuracy was poor on short casual French. Upgraded to `medium` — much better.
- Added 🎤 emoji during transcription so user knows the system is working before 🤔 appears.
- Added transcription message in Discord thread so user can see what was heard.
- Added duration to both structured log and Discord message.

## Test results

- French voice messages transcribed accurately with medium model
- Transcription takes ~2-5s on CPU for typical voice messages
- First run downloads ~1.5GB model, cached after that
- Regular text messages unaffected
- Error handling works — transcription failure doesn't crash the bot
- 42 unit tests passing
