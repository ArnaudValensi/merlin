# Session 2 — 2026-02-22

## What was done

### Bug fixes from session 1 known issues

1. **Horizontal scroll in diff view** — Added `width: max-content; min-width: 100%` to `.diff-table` so the table uses its intrinsic width and overflows the `.diff-table-scroll` wrapper. Added `overscroll-behavior-x: contain` to prevent scroll chaining. Kept `overflow: hidden` on `.diff-file-section` to prevent the whole page from scrolling horizontally (an earlier attempt with `overflow: visible` caused that).

2. **Edge-to-edge mobile layout** — Used `.main:has(#commits-app) { padding: 0 }` in the mobile media query to zero out the parent container padding for the commits page. Set `gap: 0` on `.diff-content` so file sections stack flush. Headers keep `0 8px` padding.

3. **Font size not matching terminal** — Changed all code tables from 13px/1.4 to 11px/1.2, matching the terminal's xterm.js config. Key discovery: mobile browsers (Chrome, Safari) apply **text autosizing** that boosts font sizes on wide columns. The fix was `-webkit-text-size-adjust: 100%; text-size-adjust: 100%` on the tables. Playwright can't detect this — computed styles show 11px but the browser renders larger.

4. **Font family consolidation** — Replaced all `'SF Mono', 'Fira Code', monospace` references across the entire dashboard with `'Geist Mono', monospace`. Loaded Geist Mono from Google Fonts CDN alongside Geist in base.html. 8 files updated (dashboard.css, commits.css, notes.css, terminal.html, base.html, login.html, 2 docs).

### Infrastructure improvements

5. **No-cache middleware** — Added `NoCacheStaticMiddleware` to `dashboard.py` that sets `Cache-Control: no-store` on all `/static/` responses. Eliminates stale CSS issues during development.

6. **FAB navigation: hunks instead of lines** — Changed the file view FAB (up/down arrows) to jump between hunks (groups of consecutive changes) instead of every individual changed line. Reduces gutterLines to hunk starts after collection.

## Key learnings

- **text-size-adjust** is critical for any mobile view with small code fonts. Mobile browsers boost wide-column text for readability. Playwright doesn't simulate this.
- **Browser caching** of static files caused multiple rounds of "it didn't change" feedback. The no-cache middleware solves this permanently for dev.
- **Geist Mono** is available on Google Fonts and can be loaded in the same request as Geist using `&family=Geist+Mono:wght@400;500;700`.
