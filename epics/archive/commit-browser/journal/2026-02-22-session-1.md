## Known issues — to fix next session

### 1. Horizontal scroll still broken
The diff table scroll wrapper (`div.diff-table-scroll`) was added but the **structure is wrong**. The file header and the diff table are still siblings inside the same container. What's needed:

**Current structure:**
```
.diff-file-section (overflow: hidden)
  .diff-file-header (path + Full file btn)
  .diff-table-scroll (overflow-x: auto)
    .diff-table
```

This should work in theory but the overflow-x on `.diff-table-scroll` isn't taking effect. Need to debug — likely the table needs `width: max-content` or the scroll wrapper needs explicit constraints. Test by inspecting in browser DevTools.

### 2. Still has side padding/margins on mobile
The edge-to-edge CSS changes aren't fully effective. There are still margins/paddings coming from somewhere — possibly from `.commits-view`, the `.main` container, or the `.diff-content` flex gap. Need to audit the full CSS cascade on mobile and eliminate all horizontal spacing for code views.

### Design direction: terminal-like full-width layout
User wants the diff/file views to feel like a **terminal or tmux** — code takes the full viewport width, no decorative padding. This may require restructuring the page layout so that in diff/file views:
- The view fills the full width below the sidebar
- No padding on the main content area
- Code blocks are truly flush with screen edges
- Only the header (back button, file name) gets minimal padding

This is a more fundamental layout change than just tweaking margins. Consider making diff/file views use a different layout class that overrides the `.main` padding entirely.
